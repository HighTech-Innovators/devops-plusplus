# Contributing to DevOps++

We welcome contributions from everyone. Here's how you can help:

1. **Add content**: Write about DevOps++ principles, playbooks, or case studies in Markdown.
2. **Improve examples**: Add or refine Kubernetes, CI/CD, or observability code snippets.
3. **Fix docs**: Improve clarity, fix typos, or add diagrams in `/images`.

## Workflow
- Fork this repo
- Create a branch (`feature/my-improvement`)
- Commit changes with clear messages
- Submit a Pull Request

## Code of Conduct
We aim for an open, inclusive community. Please be respectful and constructive.
